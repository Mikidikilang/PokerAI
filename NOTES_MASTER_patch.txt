5. OPPONENT HUD TRACKER (core/opponent_tracker.py)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

RÉGI (v3.x): nyers action count → frequency vector
  → 7 dim per player (action 0-6 frekvenciája)
  PROBLÉMA: nem ad igazi poker statisztikákat

ÚJ (v4): valódi HUD statisztikák per player
  VPIP  = preflop call/raise / preflop opportunities
  PFR   = preflop raise / preflop opportunities
  AF    = (bet+raise) / call (aggression factor)
  3bet% = 3bet / 3bet_opportunity
  fold_to_3bet% = fold when facing 3bet / times faced 3bet
  cbet% = continuation bet flop / flop seen as preflop aggressor
  fold_to_cbet% = fold to cbet / times faced cbet

  Per player: 7 stat × num_players dim
  memory_length = 1000 action (rolling window)
  Cold start: 0.5 (neutral prior) – nem 0!

KÉT OSZTÁLY:

  OpponentHUDTracker (training pipeline):
    - Lokális asztali indexek alapján tárol (0..N-1)
    - A training collector ezt használja, változatlan
    - reset() minden session-nél

  GlobalPlayerTracker (RTA v4):
    - Username alapján tárol → modellváltás-biztos
    - KETTŐS RÉTEG:
        Rolling window (memória): utolsó 1000 esemény per játékos
          → adaptív, session-szintű, a modell ezt látja
          → session végén eldobódik, nem kell perzisztálni
        Lifetime számlálók (SQLite): összes valaha látott esemény int-ként
          → soha nem törlődik, session-független
          → 14 int per játékos → ~8 MB / 50k játékos
    - COLD START BLENDING:
        Ha rolling < 30 esemény: lifetime stat lineárisan bekeveredik
        Így egy régen látott ellenfél adatai azonnal hasznosak
    - SKÁLÁZHATÓSÁG:
        50k játékos  → ~8 MB SQLite, <1ms/játékos lookup
        500k játékos → ~80 MB SQLite, azonos sebesség
        RAM          → max 9 PlayerStats él egyszerre (asztalon ülők)
    - AUTO-FLUSH: 50 akciónként írja ki a dirty lifetime értékeket SQLite-ba
    - PRELOAD: manage_table_change()-ben egyetlen batch DB query az asztalra
    - MIGRÁCIÓ: GlobalPlayerTracker.migrate_from_json('players.json', 'players.db')

KÉT OSZTÁLY:

  RealtimePokerAssistant (inference/realtime_assistant.py):
    - Egyszerű, egy modell, egy asztalmméret
    - OpponentHUDTracker (lokális index, memória csak)
    - Visszafelé kompatibilis régi kóddal
    - Ajánlott: egyszerű teszteléshez, prototípushoz

  RTAManager (inference/rta_manager.py):
    - Multi-asztalos, több modell, SQLite ellenfél DB
    - AJÁNLOTT ÉLES HASZNÁLATHOZ
    - Context manager-ként auto-flush kilépéskor

RTAManager RÉSZLETES SPEC:

  __init__(model_paths, db_path, device, equity_sims, tracker_memory)
    model_paths:   dict[int, str] → {num_players: '.pth fájl'}
    db_path:       SQLite fájl elérési útja (None = csak memória)
    → ModelPool lazy betöltéssel
    → GlobalPlayerTracker betöltése db_path-ból
    → session elején loggolja a DB méretét és játékosszámot

  manage_table_change(num_players, seat_map, my_seat, button_seat)
    seat_map: dict[int, str] → {szék_idx: username}
    1. Modell lazy betöltése a pool-ból
    2. SeatMapper frissítése (username ↔ lokális index fordítás)
    3. ActionHistoryEncoder újraalkotása (helyes num_players-szel)
    4. Action history törlése (inkompatibilis lenne a régi mérettel)
    5. GlobalPlayerTracker.preload_table() → batch DB query
    FONTOS: GlobalPlayerTracker NEM resetelődik

  new_hand(my_stack, all_stacks, bb, sb)
    all_stacks: dict[username, float]

  new_street(street)
    street: 0=preflop, 1=flop, 2=turn, 3=river

  get_recommendation(obs_vector, legal_actions, hole_cards, board_cards,
                     current_pot, call_amount)
    → dict:
        action, action_name, confidence, top3
        equity, spr, m_ratio, street_name
        value_est, explanation
        db_players  ← SQLite-ban ismert összes játékos száma

  record_opponent_action(username, action, bet_amount, pot_size, context)
    context: {'facing_3bet': bool, 'is_cbet_opp': bool, 'facing_cbet': bool}

  record_my_action(action, bet_amount, pot_size)

  flush_tracker()
    → dirty lifetime értékek → SQLite
    → context manager automatikusan hívja kilépéskor

  reset_session()
    → memória törlése, DB NEM érintett

  player_stats(username) → summary dict
  all_player_stats()     → session-szintű összes játékos
  top_players(n)         → top N lifetime kézszám szerint (DB query)
  db_info()              → DB méret, játékosszám, cache állapot

BELSŐ KOMPONENSEK:

  ModelPool:
    - dict[int, AdvancedPokerAI] → lazy loading
    - preload_all(): startup-ban hívható az összes modell előtöltéséhez

  SeatMapper:
    - seat_map frissítése: {szék_idx: username}
    - local_index(username) → int (0..N-1)
    - build_local_stacks(stack_by_username) → list (build_state_tensor-hoz)
    - as_seat_map() → dict (preload_table-hoz)

  _GlobalTrackerAdapter:
    - Bridge: GlobalPlayerTracker → get_stats_vector() interface
    - A build_state_tensor() OpponentHUDTracker.get_stats_vector()-t vár
    - Ez az adapter burkolja a SQLite-alapú trackert
